Technology

Built edge-first, not cloud-first.

SensorDyme runs entirely on Raspberry Pi 5 class hardware — no GPU, no accelerator card, no dedicated inference server. Every model executes on-CPU, at the edge, in real time.

Perception is handled by a combination of MediaPipe for landmark and pose tracking, and YOLO-nano for lightweight object detection — models chosen specifically because they run efficiently without specialized hardware.

The result is a device that processes 25–30 frames per second locally, with no dependency on network bandwidth, latency, or uptime to function.

Privacy

Privacy isn't a policy. It's the architecture.

SensorDyme is designed so that privacy is a property of how the system is built, not a promise layered on top of it.

On-device processing

Every frame is captured, analyzed, and discarded on the device itself. Inference never depends on a round trip to a server.

Landmarks, not video

Models output numerical landmarks, poses, and counts — not images. There is no video to review, export, or misplace.

No cloud dependency

The device operates fully offline. Connectivity is optional, used only to deliver the resulting events, not the source frames.
